# 证据格式规范

每个技术结论都必须内嵌证据块，读者应能仅通过证据独立验证结论。

**写作原则**：
- 📋 图标 + 加粗标题标识证据块
- 先给结论，再给证据
- 检测脚本 + 返回值成对出现
- API 证据必须包含请求体 + 响应体
- L1 级发现必须附带归属分析
- 功能描述必须附带数据流来源

## 6 种证据类型

### ① JS 检测脚本返回值

~~~markdown
📋 **证据 — {检测项}**：

```javascript
// 检测脚本
{脚本代码}
```

```json
// 返回值
{实际返回的 JSON}
```

**解读**：{一句话解释}
~~~

### ② API 请求/响应原文

~~~markdown
📋 **证据 — {API 名称}**：

```http
{METHOD} {URL}
{Headers}
{Request Body}
```

```json
// 响应
{Response Body}
```
~~~

### ③ DOM 结构

~~~markdown
📋 **证据 — {DOM 特征}**：

```
{元素层级结构，含 class/属性}
```
~~~

### ④ CDN/资源 URL

~~~markdown
📋 **证据 — {资源类型}**：
- {完整 URL 1}
- {完整 URL 2}
~~~

### ⑤ Network 面板抓包

~~~markdown
📋 **证据 — {请求类别}**：

| reqid | URL | 状态 |
|:---|:---|:---|
| | | |
~~~

### ⑥ 数据获取链路

~~~markdown
📋 **证据 — {链路名称}**：

| 步骤 | 接口/资源 | 返回关键数据 |
|:---|:---|:---|
| | | |
~~~
